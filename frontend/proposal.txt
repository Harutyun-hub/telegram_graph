================================================================================
  1. EXECUTIVE SUMMARY
================================================================================

Radar Obshchiny is a comprehensive intelligence platform designed to monitor,
analyze, and extract actionable insights from Armenian Telegram communities.
The platform processes thousands of Telegram messages daily through a GPT-4o-mini
extraction pipeline, structures the output into a Neo4j knowledge graph with
12 node types and 16 relationship types, and presents insights through a
multi-page admin dashboard with 30+ analytical widgets.

The platform serves three distinct user personas:

  - EXECUTIVES who need 30-second community pulse summaries and AI-generated
    briefings to inform strategic decisions without digging into raw data.

  - INTELLIGENCE ANALYSTS who conduct deep-dive investigations into topics,
    sentiment patterns, and network structures across channels and time periods.

  - EXPERT RESEARCHERS who require granular access to raw data, graph
    exploration, Cypher query capabilities, and AI-assisted querying.

Key metrics at a glance:

  - Daily messages processed:       14,000+
  - Telegram channels monitored:    50+
  - Dashboard analytical widgets:   30+
  - Graph node types:               12
  - Graph relationship types:       16
  - Supported languages:            Russian, English, Armenian


================================================================================
  2. PROBLEM STATEMENT
================================================================================

Armenian communities in Telegram have grown into massive, fragmented ecosystems
-- dozens of channels covering housing, employment, education, legal matters,
social integration, and daily life. Community leaders and organizations face
critical challenges:

INFORMATION OVERLOAD
  Thousands of messages per day across 50+ channels make manual monitoring
  impossible. No single person or team can read and categorize everything.

NO STRUCTURED INTELLIGENCE
  Critical community needs, sentiment shifts, and emerging problems are buried
  in unstructured chat data with no way to systematically extract or track them.

DISCONNECTED INSIGHTS
  Each channel operates in a silo -- cross-channel patterns (e.g., housing
  complaints correlating with neighborhood discussions) are invisible without
  graph-based analysis.

DELAYED RESPONSE
  By the time community leaders manually identify a trending problem (e.g., a
  scam affecting members), the damage has already spread across multiple
  channels.

NO AUDIENCE UNDERSTANDING
  Community organizations lack demographic insights, persona profiling, and
  engagement funnel data needed for strategic planning and resource allocation.


================================================================================
  3. SOLUTION OVERVIEW
================================================================================

Radar Obshchiny solves these challenges through a four-layer intelligence
architecture:

LAYER 1: TELEGRAM LISTENER
  Real-time ingestion from 50+ channels via Telegram Bot API / Telethon, with
  deduplication, queue management (Redis), and automatic channel discovery.

LAYER 2: AI EXTRACTION ENGINE
  GPT-4o-mini processes each message batch, extracting entities, topics,
  sentiment, problems, questions, and recommendations using optimized prompt
  chains with structured JSON output.

LAYER 3: KNOWLEDGE GRAPH (Neo4j)
  All extracted intelligence is stored as an interconnected graph with 12 node
  types and 16 relationship types, enabling relationship discovery, pattern
  analysis, and graph algorithm execution.

LAYER 4: INTELLIGENCE DASHBOARD
  React-based admin dashboard with 30+ widgets across 8 analytical tiers,
  force-directed graph visualization, and an AI-powered conversational
  assistant for natural language querying.

================================================================================
  5. TELEGRAM INGESTION PIPELINE
================================================================================

ARCHITECTURE
-------------------------------------------------------------------------------

The ingestion layer uses Telethon (Python) for real-time Telegram monitoring
with fallback to the Telegram Bot API. Messages are buffered in a Redis queue
for reliable, ordered processing.

  - Multi-channel listener: Monitors 50+ Armenian community channels
    simultaneously with configurable priority levels.

  - Message deduplication: Content-hash based dedup prevents processing
    forwarded/cross-posted messages multiple times.

  - Rate limiting: Respects Telegram API limits with exponential backoff and
    circuit breaker patterns.

  - Channel auto-discovery: Detects new relevant channels from forwarded
    messages and channel mentions.

  - Historical backfill: Can ingest up to 6 months of historical data per
    channel for initial graph population.

  - Media handling: Extracts text from images (OCR), voice messages
    (Whisper API), and document attachments.

PROCESSING PIPELINE
-------------------------------------------------------------------------------

  Raw Message -> Language Detection -> Text Cleaning -> Queue (Redis) ->
  Batch Formation -> GPT-4o-mini -> Validation -> Neo4j Write

ESTIMATED THROUGHPUT
-------------------------------------------------------------------------------

  Messages / Day:         14,000+
  Processing Latency:     < 30 seconds
  API Cost / Day:         ~$2.50


================================================================================
  6. AI EXTRACTION ENGINE (GPT-4o-mini)
================================================================================

EXTRACTION PIPELINE
-------------------------------------------------------------------------------

Each message batch is processed through a carefully engineered GPT-4o-mini
prompt chain that extracts structured data in a single API call, optimizing
for cost efficiency while maintaining extraction quality.

EXTRACTED DATA TYPES
-------------------------------------------------------------------------------

  Topics
    Auto-classified into 20+ category taxonomy (housing, banking,
    education, legal, employment, social, healthcare, etc.)

  Sentiment
    Tri-polar classification (positive/negative/neutral) with intensity
    score 0-10

  Named Entities
    People, organizations, locations, products, services, prices

  Problems
    Community pain points with severity (1-10) and urgency (1-10) scoring

  Questions
    Extracted questions with answer-status tracking

  Recommendations
    User-generated recommendations with confidence score and category

  Demographics
    Inferred user attributes: origin city, age range, gender, profession

  Relationships
    16 relationship types connecting all extracted entities in the graph

COST OPTIMIZATION STRATEGY
-------------------------------------------------------------------------------

  - Batch processing: 50-100 messages per API call reduces overhead tokens
    by ~60%.

  - Structured output mode: JSON schema enforcement eliminates parsing
    errors and retries.

  - Tiered processing: Simple messages get lightweight extraction; complex
    threads get deep analysis.

  - Estimated monthly API cost: ~$75-$100 for GPT-4o-mini at 14K
    messages/day.


================================================================================
  7. GRAPH DATABASE (Neo4j)
================================================================================

GRAPH SCHEMA -- 12 NODE TYPES
-------------------------------------------------------------------------------

  1.  User            Telegram user profile with demographics, activity
                      metrics, persona classification

  2.  Channel         Telegram channel/group with metadata, member count,
                      activity level

  3.  Message         Individual message with extracted entities, sentiment,
                      language

  4.  Topic           Auto-detected topic cluster (Housing, Education,
                      Banking, etc.)

  5.  Sentiment       Emotional classification node (positive, negative,
                      neutral, mixed)

  6.  Entity          Named entity (person, organization, location, product)

  7.  Problem         Community pain point with severity and urgency scoring

  8.  Question        Extracted question with answer status tracking

  9.  Recommendation  User-generated recommendation (service, place, product)

  10. Event           Community event mention with date, location, type

  11. Location        Geographic reference (neighborhood, city, country)

  12. Interest        User interest cluster for psychographic profiling

16 RELATIONSHIP TYPES
-------------------------------------------------------------------------------

  POSTED_IN          AUTHORED           MENTIONS_TOPIC     HAS_SENTIMENT
  REPLIES_TO         MEMBER_OF          REFERS_TO_ENTITY   RAISES_PROBLEM
  ASKS_QUESTION      RECOMMENDS         LOCATED_IN         INTERESTED_IN
  SIMILAR_TO         INFLUENCES         ANSWERED_BY        TRIGGERED_BY

GRAPH ALGORITHMS USED
-------------------------------------------------------------------------------

  PageRank
    Identifies most influential community members across the network.

  Community Detection (Louvain)
    Discovers clusters of users with shared interests and topics.

  Shortest Path
    Traces information spread patterns between channels and users.

  Similarity (Jaccard)
    Recommends related topics and identifies duplicate discussions.

================================================================================
  9. GRAPH VISUALIZATION MODULE
================================================================================

The graph visualization page provides an interactive, force-directed graph
explorer built with react-force-graph-2d, enabling analysts to visually explore
the knowledge graph, discover hidden relationships, and investigate network
patterns.

FEATURES
-------------------------------------------------------------------------------

  - Force-directed layout: Physics-based node positioning with adjustable
    charge, distance, and collision parameters.

  - Node Inspector panel: Click any node to see full details, connected
    entities, and relationship paths.

  - Dynamic filtering: Filter by node type, sentiment, topic, channel, time
    range -- graph updates in real-time.

  - Visual clustering: Color-coded node types with size mapped to
    influence/activity metrics.

  - Search & focus: Search for any entity and auto-zoom to its neighborhood
    in the graph.

  - Export capabilities: Export visible graph as PNG, SVG, or raw JSON for
    external analysis.

  - Graph legend: Interactive legend showing all node and relationship types
    with toggle visibility.

  - Floating controls: Zoom in/out, zoom to fit, center graph buttons.

  - Global filters: Filter by brands/entities, sentiments, timeframe with
    real-time graph response.


================================================================================
  10. AI ASSISTANT
================================================================================

A floating conversational AI assistant integrated into the dashboard, accessible
from any page. It enables natural language querying over the Neo4j knowledge
graph, providing instant answers with source citations.

CAPABILITIES
-------------------------------------------------------------------------------

  - Natural Language -> Cypher: Translates user questions into Neo4j Cypher
    queries automatically.

  - RAG over graph data: Retrieval-Augmented Generation using graph context
    for accurate, grounded responses.

  - Contextual suggestions: Pre-built prompt suggestions based on current
    page context and recent data.

  - Streaming responses: Real-time token streaming for responsive UX.

  - Bilingual support: Responds in Russian or English based on user language
    preference.

  - Mobile: 5th tab in bottom navigation for quick access on mobile devices.

  - Conversation history: Maintains session context for follow-up questions.

EXAMPLE QUERIES
-------------------------------------------------------------------------------

  "What are the top community concerns this week?"
  "Which topics need urgent attention?"
  "Who are the most influential voices in the housing discussion?"
  "Summarize housing sentiment trends over the last month"
  "Which channels have the most cross-posting activity?"
  "How many new members joined community channels this week?"

AUTHENTICATION
-------------------------------------------------------------------------------

  - Supabase Auth: Email/password authentication with secure session

  - Row-Level Security: Supabase RLS policies ensure data isolation between
    organizations.

  - Session management: JWT-based with automatic refresh and secure cookie

THREE USER ROLES
-------------------------------------------------------------------------------

  EXECUTIVE
    Access to Dashboard (Tier 1-2), high-level trends, AI briefings.
    No raw data access. Read-only interface optimized for quick consumption.

  INTELLIGENCE ANALYST
    Full dashboard access (all 8 tiers), graph explorer, topic deep-dives,
    audience profiles. Can create saved views and export reports.

  EXPERT RESEARCHER
    Everything + raw Cypher queries, API access, export capabilities,
    pipeline configuration, and user management.

MULTI-LANGUAGE SUPPORT
-------------------------------------------------------------------------------

  Russian (default):  Primary language for the Armenian-Russian diaspora
                      community. Set as default in LanguageContext.

  English:            Full translation for international stakeholders and
                      partners.

  Armenian:           Native language support for local community leaders.

  Implementation:     LanguageContext.tsx provider propagates locale to all
                      components via useLanguage() hook. Each widget renders
                      content based on lang === 'ru' conditional.

  16. RISK MITIGATION
================================================================================

  RISK: Telegram API Rate Limits
  SEVERITY: Medium
  MITIGATION: Exponential backoff, circuit breaker, multiple bot tokens,
  Telethon session pooling. Fallback to less frequent polling during
  high-load periods.

  RISK: GPT-4o-mini Output Quality
  SEVERITY: Medium
  MITIGATION: Structured output schema with JSON validation, fallback
  prompts for edge cases, human-in-the-loop review for training data
  refinement. Continuous prompt tuning based on extraction accuracy metrics.

  RISK: Neo4j Performance at Scale
  SEVERITY: Low
  MITIGATION: Composite indexes on high-query properties, query optimization
  with EXPLAIN/PROFILE, pagination for large result sets, archival strategy
  for data older than 6 months.

  RISK: Scope Creep
  SEVERITY: High
  MITIGATION: Fixed-scope phases with clear deliverables, MoSCoW
  prioritization (Must/Should/Could/Won't), bi-weekly stakeholder demos,
  formal change request process with impact assessment.

  RISK: Data Privacy (GDPR)
  SEVERITY: Medium
  MITIGATION: Only public Telegram data is processed. No private messages.
  No PII storage -- usernames are pseudonymized. Anonymization layer for
  exports. Data retention policies with automatic purging. Supabase RLS
  for access control.

  RISK: Team Availability
  SEVERITY: Low
  MITIGATION: Overlapping knowledge between team members (bus factor >= 2),
  comprehensive documentation, modular architecture allowing independent
  component development.
